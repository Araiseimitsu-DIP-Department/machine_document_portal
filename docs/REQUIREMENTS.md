# 要件整理

## 確定事項

- プロジェクト名は Machine Document Portal、フォルダ名は `machine_document_portal` とする。
- Googleスプレッドシートの「稼働中工程内検査シート」をWebアプリへ置き換える。
- 号機ごとに現在生産中の品番と品名を表示する。
- 品番はGoogleスプレッドシート「生産中」から取得する。
- 工程内検査シートはSharePoint、加工図はNASに存在する。
- 各号機に2種類の資料ボタンと取得状態を表示する。
- 表示対象は「生産中」シートのD列に存在する号機とし、追加・削除は次回更新時に反映する。
- 号機はグループ、数値の号機番号の順に並べる。
- 当面はPostgreSQLへ保存せず、アプリのプロセス内メモリで状態を保持する。
- SQLiteは使用しない。
- PostgreSQL構成は将来の選択肢として残すが、`PERSISTENCE_MODE=memory` では接続しない。
- UIは `docs/DESIGN` のガイド、ロゴ、ブランド配色を参考にする。
- Windows業務PCと工場内タブレットで利用する。
- 本番ではGoogle Sheets APIへ読み取り専用で接続し、サンプルデータは画面確認用とする。

## 今回の仮仕様

- 号機グループ、台数、色、サンプル状態は `sample_data/machines.json` で変更できる。
- サンプルデータの既定号機は「生産中・両資料あり」とし、A-1～B-3で全代表状態を確認する。
- 号機カードはグループごとの可変グリッドとし、画面幅に応じて列数を変える。
- 工程内検査シートと加工図は別々の状態を持つ。
- 品番正規化はNFKC、大文字化、空白除去、ハイフン統一を行い、枝番・改訂番号・アンダースコアは保持する。
- 加工図は `NAS_DRAWING_DIRECTORY` 配下でH列の品番と同名のPDFを検索し、暫定的に1ページ目をアプリ内で画像表示する。
- 日時はタイムゾーン付きで扱い、UIでは現地時刻の文字列表現を行う。
- UIの最終更新時刻は日本時間（Asia/Tokyo）で表示する。
- 生産状態はA列から取得し、稼働中／停止中／生産終了／セット中をそれぞれ緑／赤／紫／青で表示する。
- 号機、品番、資料リンク、取得状態はプロセス内メモリに保持し、アプリ再起動時に消える。
- 同一品番の資料検索結果はメモリ内で再利用する。
- メモリ共有を行わないため、当面のUvicorn起動は1ワーカーとする。
- アプリは `AUTO_REFRESH_SECONDS` 秒ごとにGoogle Sheetsを定期同期し、ブラウザはメモリ上の最新状態を再読込する。`0`で自動更新を無効にする。
